# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✅ Yes    |

## Reporting a Vulnerability

If you discover a security vulnerability, **do not open a public GitHub issue.**

Email the maintainer directly. You will receive a response within 72 hours. If the vulnerability is confirmed, a patch will be released within 14 days.

## What ARIA Stores

ARIA stores no personal data. All data processed by the framework is either:
- **Simulated demo data** generated by `core/data_simulator.py` — synthetic, no real individuals
- **Your own operational data** from your Gold layer CSVs — stored locally, never transmitted externally

Runtime files written to `data/` (event bus, logs, approval queue) contain only:
- Domain entity names and business values you configure
- Timestamps and module status events
- No PII, no credentials, no payment data

## API Key Handling

- The `ANTHROPIC_API_KEY` is read from `.env` via `os.getenv()` — never hardcoded
- The `ARIA_MCP_TOKEN` is read from the environment — never stored in config files
- SMTP credentials for notifications are read **exclusively** from `ARIA_SMTP_USER` / `ARIA_SMTP_PASSWORD` env vars — never set them in `aria_config.yaml`

The `.env` file is in `.gitignore` and must never be committed.

## MCP Server

The MCP server (`mcp/aria_mcp_server.py`) binds to `127.0.0.1` (localhost only) by default.

- In production, set `ARIA_MCP_TOKEN` to a strong random string to enable Bearer token auth
- To bind to all interfaces, set `ARIA_MCP_HOST=0.0.0.0` — only do this behind a firewall or reverse proxy
- Do not expose port 8765 to the public internet without authentication
- For zero-trust environments, run the MCP server behind a reverse proxy with TLS

## Demo Mode

`aria_config.yaml → aria.demo_mode: true` disables all Anthropic API calls. Safe to run in offline environments or for demos without an API key.
